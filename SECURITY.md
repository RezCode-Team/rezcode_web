# Security Policy for REZCODE

At **REZCODE**, we take the security of our digital solutions and our clients' data very seriously. If you believe you have found a security vulnerability, please let us know so that we can resolve the issue promptly.

## Supported Versions

We actively provide security updates for the following versions of our official website and core components:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report vulnerabilities by following these steps:

1. **Private Report:** Send an Dm to `@rezcode.agency`.
2. **Details:** Include a detailed description of the vulnerability, steps to reproduce, and the potential impact.
3. **Timeline:** - You will receive an acknowledgment of your report within **24-48 hours**.
   - We will provide status updates every **3 days** until the issue is resolved.
   - We kindly ask for a **90-day responsible disclosure period** before any public information is shared.

## Preferred Communication

We prefer all security-related communication to be in **English** or **Bahasa Indonesia**.

## GitHub Security Advisories
We may also use [GitHub Security Advisories](https://github.com/REZCODE-Agency/repo-name/security/advisories) to privately collaborate on a fix and publish information about security vulnerabilities in this repository.

*Thank you for helping keep REZCODE and our community safe.*
